Hermes Java SDK Architecture

Layered Java SDK for Hermes Agent — 80 tests, Transport abstraction, Thread-safe

Client Layer

  • • HermesClient — main entry point
  • • ThreadSafeChatSession — thread-safe multi-turn
  • • ChatSession — basic multi-turn
  • • HermesProperties — Spring Boot config
  • • HermesController — REST adapter (optional)

SDK / Business Layer

  • • HermesSdk — fluent API facade
  • • SessionManager — CRUD + cache
  • • ToolsetManager — toolset query + cache
  • • SkillExecutor — async skill execution
  • • SystemMonitor — health / capabilities

Core / Transport

  • • HermesApi — raw API (1:1 gateway)
  • • Transport interface — pluggable
  • • HttpTransport — OkHttp (✓ impl)
  • • RpcTransport — gRPC (○ future)
  • • WebSocketTransport — SSE (○ future)

Exception Hierarchy

  • • HermesException — base (errorCode + status)
  • • HermesNetworkException — network errors (-1)
  • • HermesApiException — API errors (4xx/5xx)
  • • HermesAuthException — auth errors (401)

Testing Coverage

  • • Config (4) + Exception (6) + DTO (7)
  • • Client (11) — builder, validation
  • • SessionService (11) + RunService (8)
  • • SkillService (11) — mock HermesClient
  • • Security (14) — injection defense
  • • Performance (8) — timeout, concurrency
  • • Total: 80 tests, all passing

Key Design Decisions

  • • No apiKey/model — baseUrl only
  • • No Lombok — manual getter/setter
  • • Transport abstraction — hot-swap
  • • Immutable config (final fields)
  • • CompletableFuture for async
  • • Log4j2 over Logback (2.24.1)
